DA - Delivery Truck Pallet Packing Optimization
A delivery truck packing optimization system that maximizes profit while respecting weight constraints using classic and modern algorithms. This system reads pallet and truck data from CSV files and applies a variety of optimization techniques—including Brute Force, Dynamic Programming, Greedy Approximation, and Integer Linear Programming—to solve a knapsack-like problem. It provides an interactive menu interface for selecting datasets and algorithms, making it an educational and practical tool for analyzing different algorithmic strategies in combinatorial optimization.

Calculates the upper bound of profit from a given node using LP-relaxation.

This function estimates the best possible profit that can be obtained starting from the current node. It greedily adds pallets to the capacity until the truck is full or the next pallet would exceed the capacity. If a pallet can't fully fit, a fractional part of its profit is added to represent an optimistic upper bound.

Parameters
nodeThe current decision node in the Branch and Bound tree.
truckCapacityThe maximum allowable weight of the truck.
palletsThe list of all available pallets sorted by profit-to-weight ratio.
Returns
An integer representing the estimated maximum profit achievable from this node.

void ILPBranchAndBound::solve ( const Truck & truck)

Solves the knapsack problem using Integer Linear Programming with a Branch and Bound strategy.

This method applies a Branch and Bound algorithm to determine the subset of pallets that maximizes profit without exceeding the truck's weight capacity. It uses a greedy-based LP relaxation to estimate upper bounds and prunes non-promising branches to optimize the search.

Parameters
truckThe truck containing a list of pallets and its capacity.

The method sorts the pallets by profit-to-weight ratio, explores decision nodes in a BFS manner, and updates the best solution found as it traverses the tree. Each node tracks the level of decision, cumulative profit, weight, and selection path.

Complexity

  • Time Complexity: O(2^n) in the worst case, where n is the number of pallets (full tree traversal). However, pruning significantly reduces the number of evaluated nodes in practice.
  • Space Complexity: O(n) for tracking selection paths per node.
